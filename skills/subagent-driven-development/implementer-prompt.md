# 实现子代理提示模板

派发实现子代理时使用此模板。

```text
Task tool（通用任务类型）:
  description: "实现任务 N：[任务名称]"
  prompt: |
    你正在实现任务 N：[任务名称]

    ## 任务说明

    [在这里粘贴计划中该任务的**完整全文**，不要让子代理自己去读文件]

    ## 上下文

    [说明这个任务在整体方案中的位置、依赖关系、架构背景]

    ## 开始前

    如果你对以下内容有疑问：
    - 需求或验收标准
    - 实现思路或技术方案
    - 依赖关系或前置假设
    - 任务描述里任何不清楚的地方

    **现在就提出问题。** 在开始动手之前，把疑问和风险都提出来。

    ## 你的职责

    当你已经明确需求后：
    1. 严格按任务要求实现
    2. 编写测试（如果任务要求 TDD，就遵循 TDD）
    3. 验证实现确实可用
    4. 提交本次变更
    5. 自查代码（见下方）
    6. 汇报结果

    工作目录：[directory]

    **执行过程中：** 如果遇到任何意外情况或不明确之处，**立即提问**。
    允许暂停并澄清；不要猜测，也不要自行脑补需求。

    ## 汇报前：先做自查

    用全新的眼光检查你的工作，问自己：

    **完整性：**
    - 我是否完整实现了规格里的所有内容？
    - 有没有遗漏的需求？
    - 有没有未覆盖的边界情况？

    **质量：**
    - 这是我当前能交付的最好版本吗？
    - 命名是否清晰准确（描述“它做什么”，而不是“它怎么做”）？
    - 代码是否干净、可维护？

    **纪律：**
    - 我是否避免了过度实现（YAGNI）？
    - 我是否只实现了被要求的内容？
    - 我是否遵循了代码库现有模式？

    **测试：**
    - 测试是否真的验证了行为，而不是只验证 mock？
    - 如果要求 TDD，我是否遵循了它？
    - 测试是否足够完整？

    如果你在自查中发现问题，请先修复，再汇报。

    ## 汇报格式

    完成后，请按下面结构汇报：
    - 已实现内容
    - 已执行的测试与测试结果
    - 变更文件
    - 自查发现的问题（如果有）
    - 仍需关注的事项或风险
```
