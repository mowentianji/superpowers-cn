# 规格符合性审查子代理模板

派发规格符合性审查子代理时，请使用此模板。

**目的：** 验证实现者做出的内容是否**刚好符合要求**，既不遗漏，也不多做。

```text
Task tool（通用任务类型）:
  description: "审查任务 N 的规格符合性"
  prompt: |
    你正在审查某项实现是否符合其规格说明。

    ## 原始要求

    [任务要求的完整文本]

    ## 实现者声称已完成的内容

    [来自实现者的汇报]

    ## 关键提醒：不要相信汇报本身

    实现者可能完成得过快，他们的汇报可能不完整、不准确，或者过于乐观。
    你**必须独立验证**所有内容。

    **不要：**
    - 仅凭对方描述就相信它已实现
    - 直接相信“已经完整实现”的说法
    - 直接接受对方对需求的解释

    **要做：**
    - 阅读他们实际编写的代码
    - 把实现与需求逐行对照
    - 检查是否遗漏了他们声称已完成的部分
    - 检查是否偷偷加入了未要求的功能

    ## 你的职责

    阅读实现代码，并验证：

    **缺失项：**
    - 是否实现了所有要求？
    - 有没有跳过或遗漏的要求？
    - 是否声称某功能可用，但实际并未实现？

    **额外项 / 不必要工作：**
    - 是否做了未被要求的内容？
    - 是否过度设计、加入了不必要的功能？
    - 是否擅自加了“顺手做的优化”而规格中并没有？

    **理解偏差：**
    - 是否误解了需求？
    - 是否解决了错误的问题？
    - 是否实现了对的功能，但方式偏离要求？

    **结论必须基于读代码，而不是基于汇报。**

    输出格式：
    - ✅ 符合规格（在你检查代码并确认全部匹配时）
    - ❌ 发现问题：[明确列出缺失项或多余项，并附 file:line 引用]
```
