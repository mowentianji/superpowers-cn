# 技能设计的说服原则

## 概述

法学硕士遵循与人类相同的说服原则。了解这种心理可以帮助您设计更有效的技能 - 不是操纵，而是确保即使在压力下也能遵循关键实践。

**研究基础：** Meincke et al. (2025) 通过 N=28,000 次人工智能对话测试了 7 条说服原则。说服技巧使遵守率提高了一倍以上 (33% → 72%, p < .001)。

## 七项原则

### 1. 权威
**它是什么：** 对专业知识、证书或官方来源的尊重。

**技能如何发挥作用：**
- 命令式语言：“你必须”、“从不”、“总是”
- 不可协商的框架：“没有例外”
- 消除决策疲劳和合理化

**何时使用：**
- 纪律执行技能（TDD、验证要求）
- 安全关键实践
- 建立最佳实践

**示例：**
```markdown
✅ Write code before test? Delete it. Start over. No exceptions.
❌ Consider writing tests first when feasible.
```

### 2. 承诺
**它是什么：** 与之前的行动、声明或公开声明保持一致。

**技能如何发挥作用：**
- 需要公告：“公告技能使用情况”
- 强制明确的选择：“选择 A、B 或 C”
- 使用跟踪：TodoWrite 作为清单

**何时使用：**
- 确保技能得到实际遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ When you find a skill, you MUST announce: "I'm using [Skill Name]"
❌ Consider letting your partner know which skill you're using.
```

### 3.稀缺性
**它是什么：** 由于时间限制或可用性有限而产生的紧迫性。

**技能如何发挥作用：**
- 有时间限制的要求：“在继续之前”
- 顺序依赖：“紧接在 X 之后”
- 防止拖延

**何时使用：**
- 立即验证要求
- 时间敏感的工作流程
- 防止“我稍后再做”

**示例：**
```markdown
✅ After completing a task, IMMEDIATELY request code review before proceeding.
❌ You can review code when convenient.
```

### 4. 社会证明
**它是什么：** 遵循他人的行为或被认为是正常的行为。

**技能如何发挥作用：**
- 通用模式：“每次”、“总是”
- 故障模式：“X 没有 Y = 故障”
- 建立规范

**何时使用：**
- 记录普遍做法
- 关于常见故障的警告
- 强化标准

**示例：**
```markdown
✅ Checklists without TodoWrite tracking = steps get skipped. Every time.
❌ Some people find TodoWrite helpful for checklists.
```

### 5.团结
**它是什么：** 共享身份、“我们”、群体归属感。

**技能如何发挥作用：**
- 协作语言：“我们的代码库”、“我们是同事”
- 共同目标：“我们都想要质量”

**何时使用：**
- 协作工作流程
- 建立团队文化
- 无等级制度的实践

**示例：**
```markdown
✅ We're colleagues working together. I need your honest technical judgment.
❌ You should probably tell me if I'm wrong.
```

### 6.互惠
**它是什么：** 返还收到的福利的义务。

**它是如何工作的：**
- 谨慎使用 - 会感到被操纵
- 很少需要的技能

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜欢
**它是什么：** 偏好与我们喜欢的人合作。

**它是如何工作的：**
- **请勿用于合规性**
- 与诚实反馈文化的冲突
- 制造阿谀奉承

**何时避免：**
- 始终遵守纪律

## 按技能类型划分的原则组合

|技能类型|使用|避免 |
|------------|-----|-------|
|纪律执行 |权威+承诺+社会证明|喜欢，互惠|
|指导/技术|适度权威+团结|重磅权威|
|协作|团结+承诺|权威、喜欢|
|参考|仅清晰度 |全部说服|

## 为什么这有效：心理学

**明确的规则减少了合理化：**
- “你必须”消除决策疲劳
- 绝对语言消除了“这是一个例外吗？”问题
- 明确的反合理化反制措施堵住了特定的漏洞

**实施意图创造自动行为：**
- 明确的触发器+所需的操作=自动执行
- “当 X 时，做 Y”比“通常做 Y”更有效
- 减少合规认知负担

**法学硕士是超人类：**
- 对包含这些模式的人类文本进行训练
- 训练数据中的权威语言优先于合规性
- 经常建模的承诺序列（陈述→行动）
- 社会证明模式（每个人都做 X）建立规范

## 道德使用

**合法：**
- 确保遵循关键实践
- 创建有效的文档
- 防止可预测的故障

**非法：**
- 操纵谋取个人利益
- 制造虚假的紧迫感
- 基于内疚的合规

**测试：** 如果用户完全理解该技术，它是否能满足用户的真正兴趣？

## 快速参考

设计技能时，询问：

1. **它是什么类型？**（纪律、指导、参考）
2. **我想改变什么行为？**
3. **适用哪些原则？**（通常是权威+纪律承诺）
4. **我是否组合了太多？**（不要使用全部七个）
5. **这符合道德吗？**（服务于用户的真正利益吗？）
